# Journalite

Obsidian 用のカレンダー・タスク管理サイドバープラグイン。

## 主な機能

### カレンダー
- 月間カレンダーを表示
- Daily ノートがある日にはドットを表示
- 日付をクリックで Daily ノートを開く（なければ作成）
- 右クリックで Daily ノートを開くメニュー

### タスク一覧
タスクを以下のカテゴリに分類して表示:
- **Overdue**: 期限切れ
- **Today**: 今日のタスク
- **Upcoming**: 2週間以内の予定
- **No Schedule**: 日付未設定の Task ノート

### エディタ連携
- 日付パターン（`@YYYY-MM-DD` など）をハイライト表示
- チェックボックスをサイドバーから直接完了

## 概念

### Daily ノート
ファイル名が `YYYY-MM-DD.md` 形式のノートは自動的に Daily ノートとして認識されます。

```
2025-01-15.md  → Daily ノート
my-note.md     → 一般ノート
```

### Task ノート
frontmatter に `done` フィールドがあるノートは Task ノートとして扱われます:

```yaml
---
done: false
startDate: 2025-01-01
dueDate: 2025-01-31
---
```

| フィールド | 説明 |
|------------|------|
| `done` | 必須。`true` で完了済み |
| `startDate` | 開始日（省略可） |
| `dueDate` | 期限日（省略可） |

`startDate` も `dueDate` もない Task ノートは「No Schedule」に表示されます。

### Task アイテム
チェックボックスの末尾に日付パターンを記述したもの:

```markdown
- [ ] タスク内容 @2025-01-15
```

#### 対応フォーマット

| パターン | 意味 |
|----------|------|
| `@2025-01-15` | 単一日付（開始日=期限日） |
| `@2025-01-15..` | 開始日のみ（以降ずっと表示） |
| `@..2025-01-15` | 期限日のみ（その日まで表示） |
| `@2025-01-01..2025-01-15` | 期間指定 |

日付パターンはエディタ内でハイライト表示されるので、正しく入力できたか視覚的に確認できます。

## インストール

1. このリポジトリを `.obsidian/plugins/journalite/` にクローン
2. `npm install` を実行
3. `npm run build` を実行
4. Obsidian の設定 → コミュニティプラグイン で Journalite を有効化

## 開発

```bash
npm install
npm run dev    # 開発モード（ファイル監視）
npm run build  # 本番ビルド
```
